План запуска ориентирован на достижение цели «запуск за 1–2 вечера». Мы исключаем лишние итерации и переходим сразу к сборке «скелета», который обеспечит жизнеспособность проекта. Основной вызов — синхронизация бота с GitHub API для работы с `catalog.json`.

`f9 c4 e8 r9 -> s7,5` 🟢

### Пошаговый план запуска (Execution Plan 9.2)

#### Шаг 1: Подготовка инфраструктуры (30–60 мин)

1. **GitHub:** Создать публичный репозиторий `brooch-catalog`.
2. **Токен:** Сгенерировать GitHub Personal Access Token (classic) с правами `repo` для бота.
3. **PythonAnywhere:** Создать бесплатный аккаунт и настроить Virtualenv (Python 3.10+).
4. **Структура:** Вручную создать через интерфейс GitHub папку `data/` и файл `data/catalog.json` с пустым списком `{"items": []}`.

#### Шаг 2: Разработка бота (Вечер 1)

1. **Инициализация:** Написать базовый код на `aiogram`.
2. **GitHub Client:** Реализовать функцию `update_catalog()`, которая:
* Берет `sha` файла `catalog.json` через GET запрос.
* Отправляет PUT запрос с обновленным JSON и этим `sha`.
* Использует `asyncio.Lock()` для защиты от одновременных заказов.


3. **Админка:** Сделать одну кнопку «Добавить товар», которая последовательно запрашивает: Название -> Цена -> Описание -> Фото.
4. **Уведомления:** Настроить отправку сообщения в личку тебе и менеджеру при каждом заказе.

#### Шаг 3: Витрина и Деплой (Вечер 2)

1. **Frontend:** ✅ **Выполнено.** Создан `Сайт/index.html` со сложным JS-скриптом, который реализует:
    - Асинхронную загрузку каталога из `catalog.json`.
    - Интерактивную галерею для каждого товара с автопрокруткой, навигацией (стрелки, точки, клавиатура) и полноэкранным режимом.
    - Адаптивный дизайн с использованием Tailwind CSS.
2. **GitHub Pages:** ✅ **Выполнено.** Включено отображение из папки `/site`.
3. **Деплой бота:** Загрузить код бота на PythonAnywhere, прописать Environment Variables и запустить.
4. **Тест:** Добавить первый товар через бота и проверить, появился ли он на сайте через 5–10 минут.
