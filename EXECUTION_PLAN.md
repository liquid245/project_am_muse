# Project AM Muse

**Technical Design Document (TDD) — версия 9.1**  
**Project AM Muse**

**Дата:** 17 марта 2026  
**Разработчик:** Georgy  
**Цель:** запустить за 1–2 вечера, минимум кода, только кнопки, один репозиторий, 0 руб/мес.

### 1. Структура репозитория (один репозиторий — brooch-catalog)

```
brooch-catalog/
├── data/                     ← все данные
│   ├── catalog.json          ← единственный источник правды (все товары + статусы)
│   └── images/               ← все фотографии
├── Сайт/                     ← движок сайта (HTML + Tailwind + JS)
├── Бот/                      ← движок бота (Python + aiogram)
├── docs/                     ← документация (TDD, EXECUTION_PLAN и т.д.)
└── README.md
```

### 2. Формат catalog.json (бот полностью управляет файлом)

```json
{
  "items": [
    {
      "id": "brooch-001",
      "title": "Брошь 'Морской бриз'",
      "description": "Ручная работа из японского бисера...",
      "price": 1250,
      "stock": 3,
      "status": "available",          // available / sold / saved
      "created_at": "2026-03-17",
      "updated_at": "2026-03-17",
      "images": ["brooch-001-01-main.jpg", "brooch-001-02-side.jpg"]
    }
  ]
}
```

Статус товара хранится **внутри JSON** (поле `status`).  
Фотографии лежат в `data/images/` и указываются по имени файла.

### 3. Сортировка
- По полю `updated_at` (самые свежие изменения сверху).
- Админ может «поднять» товар кнопкой — бот обновляет `updated_at` на текущее время.

### 4. Хостинг
- **Бот** — PythonAnywhere (Beginner free).
- **Сайт** — GitHub Pages (из папки `Сайт/`).
- Всё бесплатно навсегда.

### 5. Клавиатуры (только кнопки — никаких команд)

**Для покупателя (постоянно внизу):**
- 🛍 Каталог
- 💬 Написать менеджеру

**Под каждой карточкой товара:**
- Заказать эту брошь

**Для админа (появляется автоматически по user_id):**
- Добавить товар
- Список товаров
- Активные заказы
- Готово

### 6. Процесс заказа
- Покупатель нажимает «Заказать эту брошь».
- Бот собирает данные (ФИО, телефон, адрес).
- Уменьшает `stock`, меняет `status` на "sold" при необходимости.
- Сохраняет заказ в `data/orders.json` (или отдельный файл в data/).
- Сразу отправляет уведомление **тебе и менеджеру** в личку.

### 7. Уведомления
- Только при новом заказе (никаких ежедневных рассылок).
- Сообщение приходит и тебе, и менеджеру.

### 8. Что убрали для максимальной простоты
- Нет папок на каждый товар
- Нет item.yaml
- Нет position / сортировки кнопками ↑↓
- Нет cache-busting
- Нет Render + UptimeRobot
- Нет ежедневных напоминаний
- Нет сложных клавиатур

### 9. Non-Functional (простые)
- Стоимость: 0 руб/мес
- Ответ бота: до 3 секунд
- Максимум 300 товаров
- Нагрузка: до 20 заказов в день на старте

### 10. Risks (честно)
- PythonAnywhere может упасть — одна кнопка Restart.
- Сайт обновляется через 5–10 минут (кэш GitHub).
- Если бот упадёт — покупатели не смогут заказать до перезапуска.

### 11. Environment Variables
- `BOT_TOKEN`
- `GITHUB_TOKEN`
- `ADMIN_USER_ID`
- `MANAGER_USER_ID`

### 12. Acceptance Criteria
- Покупатель взаимодействует только через кнопки.
- Новый товар добавляется полностью через бота.
- Статус и stock управляются в `catalog.json`.
- При заказе приходят уведомления тебе и менеджеру.
- Сайт читает один `catalog.json`.
